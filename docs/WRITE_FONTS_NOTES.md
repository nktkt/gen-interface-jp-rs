# write-fonts 0.42 — API notes for hmtx / glyf / cmap / GPOS / GSUB

Reference for the implementation agents. Sourced directly from
`~/.cargo/registry/src/index.crates.io-*/write-fonts-0.42.0/`. All struct shapes
quoted below come from the autogenerated `generated/generated_*.rs` files; the
wrappers in `src/tables/*.rs` use `include!()` to splice them in.

---

## 0. Mental model

`write-fonts` mirrors `read-fonts`: every table/record/enum exists in both
crates, but the `read-fonts` types are zero-copy views over `&[u8]` while the
`write-fonts` types are plain owned `Vec`/struct/enum types you can mutate.

Two‑phase pipeline:

1. **Read** with `read_fonts::FontRef::new(bytes)` (re-exported by
   `skrifa::FontRef`, since skrifa depends on read-fonts).
2. **Convert to owned** with the `ToOwnedTable` / `ToOwnedObj` blanket impls
   (from `write_fonts::from_obj`). These exist because every `write_fonts`
   table implements `FromObjRef<read_fonts_equivalent>` /
   `FromTableRef<read_fonts_equivalent>`.
3. **Mutate** the owned struct as a normal Rust value.
4. **Compile** with `write_fonts::dump_table(&table) -> Result<Vec<u8>, Error>`,
   *or* register it on a `FontBuilder` and call `.build()`.

Key trait imports (typical):

```rust
use write_fonts::{
    FontBuilder, dump_table,
    from_obj::{ToOwnedTable, ToOwnedObj},
};
use write_fonts::read::{FontRef, TableProvider};   // requires the "read" feature
// or, if you depend on read-fonts directly:
use read_fonts::{FontRef, TableProvider};
```

There is *no* in-place mutation of the original font bytes; you always rebuild.

## 1. `FontBuilder` — top-level assembler

File: `src/font_builder.rs`.

```rust
pub struct FontBuilder<'a> {
    tables: BTreeMap<Tag, Cow<'a, [u8]>>,
}

impl<'a> FontBuilder<'a> {
    pub fn new() -> Self;                                    // == default()
    pub fn add_table<T>(&mut self, table: &T) -> Result<&mut Self, BuilderError>
    where T: FontWrite + Validate + TopLevelTable;           // compiles+validates

    pub fn add_raw(&mut self, tag: Tag, data: impl Into<Cow<'a,[u8]>>) -> &mut Self;
    pub fn copy_missing_tables(&mut self, font: FontRef<'a>) -> &mut Self;
    pub fn contains(&self, tag: Tag) -> bool;
    pub fn ordered_tags(&self) -> Vec<Tag>;
    pub fn build(&mut self) -> Vec<u8>;                      // sfnt with directory
}
```

Notes:

- `add_table` only accepts top-level tables (those that implement
  `TopLevelTable`, providing `const TAG`). It internally calls
  `dump_table(table)` and wraps any error in `BuilderError { tag, inner }`.
- `add_raw` skips compilation; useful for tables you do not want to touch
  (forward them as-is from the source font).
- `copy_missing_tables` is the canonical "patch a few tables, keep the rest"
  pattern. Iterates the source font's directory and inserts every tag that
  isn't already present.
- `build` writes the SFNT header, the table directory (sorted by tag for
  binary search), and recomputes both per-table checksums and
  `head.checksum_adjustment` automatically. The `head` data you put in is
  rewritten with the correct adjustment as long as the table is at least
  `HEAD_CHECKSUM_END = 12` bytes.
- Output table order is the OpenType-recommended order
  (`RECOMMENDED_TABLE_ORDER_TTF` for non-CFF, `RECOMMENDED_TABLE_ORDER_CFF`
  otherwise). Unknown tags sort lexicographically; `DSIG` is forced last.
- `build` consumes the tables (`self.tables.remove`) — call it last.

`dump_table` (in `src/write.rs`) is the lower-level building block:

```rust
pub fn dump_table<T: FontWrite + Validate>(table: &T) -> Result<Vec<u8>, Error>;
```

It validates first, then runs the table-packer ("hb-repacker"-style ordering of
subtables) and serializes. Use this when you want raw bytes for any table
(top-level *or* subtable) without the SFNT wrapper.

## 2. `OffsetMarker` / `NullableOffsetMarker`

File: `src/offsets.rs`.

```rust
pub struct OffsetMarker<T, const N: usize = 2> { /* obj: T */ }
pub struct NullableOffsetMarker<T, const N: usize = 2> { /* obj: Option<T> */ }
```

Both deref to `T` / `Option<T>`. Construct with `OffsetMarker::new(value)` or
`value.into()`; mutate via `DerefMut`. They wrap the things that the spec
serializes as Offset16/Offset32 (the const generic `N` is the byte width).
You do **not** compute offsets yourself; the table writer resolves them.

Common pattern when reaching into a layout subtable:

```rust
gpos.lookup_list.as_mut().lookups.push(my_lookup);   // Deref through OffsetMarker
```

`NullableOffsetMarker::as_mut()` returns `Option<&mut T>` for absent fields
(e.g. `feature_variations`).

## 3. `hmtx`

File: `src/tables/hmtx.rs` (+ `generated/generated_hmtx.rs`).

### Shape

```rust
pub struct Hmtx {
    pub h_metrics: Vec<LongMetric>,           // one per "long" metric glyph
    pub left_side_bearings: Vec<i16>,         // mono-advance tail glyphs
}

pub struct LongMetric {
    pub advance: u16,
    pub side_bearing: i16,
}
```

`Hmtx: TopLevelTable` (`TAG = "hmtx"`), `Default`, `Clone`. `LongMetric: Default`.

### Construct

```rust
// from scratch:
let mut hmtx = Hmtx {
    h_metrics: vec![LongMetric { advance: 600, side_bearing: 50 }; num_glyphs],
    left_side_bearings: Vec::new(),
};

// from skrifa/read-fonts (read-fonts hmtx requires num_h_metrics from hhea):
use write_fonts::from_obj::ToOwnedTable;
let read_hmtx: read_fonts::tables::hmtx::Hmtx = font.hmtx()?;
let mut hmtx: write_fonts::tables::hmtx::Hmtx = read_hmtx.to_owned_table();
```

`read-fonts`'s `TableProvider::hmtx()` already passes the right
`number_of_h_metrics`. You only need the manual `Hmtx::read(data, n)` form when
you have a raw byte slice.

### Mutate

It's just two `Vec`s — index by GID, push, set fields. There is one invariant:
the count `h_metrics.len() + left_side_bearings.len()` must equal the font's
`numGlyphs`, and `h_metrics.len()` must match the value you put in
`hhea.number_of_h_metrics`. The `Hmtx::validate_impl` only checks the
`h_metrics.len() <= u16::MAX` limit; you keep the other invariants yourself.

If you change which glyphs are "long" (i.e. modify the prefix length of
`h_metrics`), update `Hhea::number_of_h_metrics` to the new
`h_metrics.len()`.

### Add to FontBuilder

```rust
builder.add_table(&hmtx)?;
builder.add_table(&hhea)?;   // remember to keep these in sync
```

## 4. `glyf` (+ `loca`)

Files: `src/tables/glyf.rs`, `src/tables/glyf/{simple,composite,glyf_loca_builder}.rs`,
`src/tables/loca.rs`.

`glyf` and `loca` are tightly coupled and **must be built together** through
`GlyfLocaBuilder`. Do not try to construct `Glyf` directly; it stores the
already-serialized bytes (`pub struct Glyf(Vec<u8>)`).

### Shape

```rust
// the per-glyph type
pub enum Glyph {
    Empty,                        // no bytes written but still gets a loca entry
    Simple(SimpleGlyph),
    Composite(CompositeGlyph),
}

pub struct SimpleGlyph {
    pub bbox: Bbox,
    pub contours: Vec<Contour>,   // Contour wraps Vec<CurvePoint>
    pub instructions: Vec<u8>,    // hinting bytecode, raw
}

pub struct CompositeGlyph {
    pub bbox: Bbox,
    components: Vec<Component>,   // private — use components()/add_component
    _instructions: Vec<u8>,
}

pub struct Component {
    pub glyph: GlyphId16,
    pub anchor: Anchor,           // re-exported from read_fonts::tables::glyf
    pub flags: ComponentFlags,
    pub transform: Transform,     // re-exported from read_fonts::tables::glyf
}

pub struct Bbox { pub x_min: i16, pub y_min: i16, pub x_max: i16, pub y_max: i16 }

pub struct ComponentFlags {       // boolean fields the user controls
    pub round_xy_to_grid: bool,
    pub use_my_metrics: bool,
    pub scaled_component_offset: bool,
    pub unscaled_component_offset: bool,
    pub overlap_compound: bool,
}
```

`CurvePoint` is re-exported from `read-fonts` (`{x,y: i16, on_curve: bool}`).

`loca`:

```rust
pub struct Loca { pub(crate) offsets: Vec<u32>, loca_format: LocaFormat }
pub enum LocaFormat { Short = 0, Long = 1 }
impl Loca {
    pub fn new(offsets: Vec<u32>) -> Self;   // computes format automatically
    pub fn format(&self) -> LocaFormat;
}
```

### Construct

From a `kurbo::BezPath` (only line + quad segments — convert cubics first):

```rust
use write_fonts::tables::glyf::SimpleGlyph;
let glyph = SimpleGlyph::from_bezpath(&path)?;     // -> Result<_, MalformedPath>
// for variable fonts:
let glyphs = SimpleGlyph::interpolatable_glyphs_from_bezpaths(&paths)?;
```

From points directly (lower level):

```rust
let mut sg = SimpleGlyph::default();
sg.contours.push(Contour::from(vec![
    CurvePoint { x: 0, y: 0, on_curve: true },
    CurvePoint { x: 100, y: 0, on_curve: true },
    // ...
]));
sg.recompute_bounding_box();
```

Composite:

```rust
let mut comp = CompositeGlyph::new(
    Component::new(GlyphId16::new(1), Anchor::Offset { x: 0, y: 0 },
                   Transform::default(), ComponentFlags::default()),
    bbox_for_first_component,
);
comp.add_component(Component::new(/*…*/), bbox_for_second);
let g: Glyph = comp.into();
```

From skrifa/read-fonts: the `read_fonts::tables::glyf::Glyph` enum implements
`FromTableRef`, so you can write `let g: Glyph = read_g.to_owned_table();`.
That said, the typical mutation path doesn't read individual glyphs — you
either rebuild the whole table or copy `glyf`+`loca` raw and only edit `hmtx`,
`cmap`, etc.

### Mutate / build the table

```rust
use write_fonts::tables::glyf::{GlyfLocaBuilder, Glyph};

let mut builder = GlyfLocaBuilder::new();
for glyph in &my_glyphs {                // any of Glyph / SimpleGlyph / CompositeGlyph
    builder.add_glyph(glyph)?;            // validates + compiles immediately
}
let (glyf, loca, loca_format) = builder.build();
```

`add_glyph` accepts anything implementing the `SomeGlyph` marker trait —
`Glyph`, `SimpleGlyph`, `CompositeGlyph`. The order is GID order; an empty
glyph contributes a zero-length entry to `loca`.

The returned `loca_format` tells you what to write into `head.index_to_loc_format`.

### Add to FontBuilder

```rust
builder
    .add_table(&glyf)?
    .add_table(&loca)?;
// and update head:
head.index_to_loc_format = loca_format as i16;
builder.add_table(&head)?;
```

If you don't intend to modify glyph outlines, the simplest thing is:

```rust
// keep glyf + loca + head untouched:
fb.copy_missing_tables(font);   // pulls glyf, loca, head bytes from the source
```

## 5. `cmap`

File: `src/tables/cmap.rs` (+ `generated/generated_cmap.rs`).

### Shape

```rust
pub struct Cmap {
    pub encoding_records: Vec<EncodingRecord>,
}

pub struct EncodingRecord {
    pub platform_id: PlatformId,                        // re-exported
    pub encoding_id: u16,
    pub subtable: OffsetMarker<CmapSubtable, WIDTH_32>, // Offset32
}

pub enum CmapSubtable {
    Format0(Cmap0), Format2(Cmap2),  Format4(Cmap4),
    Format6(Cmap6), Format8(Cmap8),  Format10(Cmap10),
    Format12(Cmap12), Format13(Cmap13), Format14(Cmap14),
}
```

There are `CmapSubtable::format_N(...)` constructors for every variant
(thin wrappers over `CmapN::new`). The structs `Cmap4 { language, end_code,
start_code, id_delta, id_range_offsets, glyph_id_array }` and
`Cmap12 { language, groups: Vec<SequentialMapGroup> }` are the ones you almost
always touch. `SequentialMapGroup::new(start_char_code, end_char_code, start_glyph_id)`.

`Cmap: TopLevelTable` (`TAG = "cmap"`), `Default`, `Clone`. Subtable formats
all have `pub fn new(...)`.

### Construct — high-level

```rust
let cmap = Cmap::from_mappings([(c1, gid1), (c2, gid2), …])?;
```

`from_mappings` takes any `IntoIterator<Item = (char, GlyphId)>`. It:

1. Sorts and dedupes.
2. Returns `Err(CmapConflict { ch, gid1, gid2 })` if one char maps to two gids.
3. Emits a format-4 BMP subtable iff any char ≤ U+FFFF.
4. Emits a format-12 full-repertoire subtable iff any char > U+FFFF.
5. Adds *both* a Unicode (platform 0) and a Windows (platform 3) encoding
   record per format. The subtable bytes are reused across platforms via the
   table-packer's deduplication.

This is the recommended path for almost every codepoint→gid table. It matches
what `fontmake`/`fonttools` produce.

### Construct — manual

```rust
let sub = CmapSubtable::format_4(/* language */ 0, end_code, start_code,
                                  id_delta, id_range_offsets, glyph_id_array);
let cmap = Cmap::new(vec![
    EncodingRecord::new(PlatformId::Windows, 1 /*BMP*/, sub.clone()),
    EncodingRecord::new(PlatformId::Unicode, 3 /*BMP*/, sub),
]);
```

Or for a format-12-only table:

```rust
let groups = vec![SequentialMapGroup::new(0x1f000, 0x1f100, 1024), /* … */];
let sub = CmapSubtable::Format12(Cmap12::new(0, groups));
```

From skrifa/read-fonts:

```rust
let cmap: Cmap = font.cmap()?.to_owned_table();
```

### Mutate

```rust
cmap.encoding_records.push(EncodingRecord::new(/* … */));
if let CmapSubtable::Format4(ref mut sub) = *cmap.encoding_records[0].subtable {
    sub.id_delta[2] += 1;    // Deref through OffsetMarker
}
```

For nontrivial edits, prefer rebuilding via `Cmap::from_mappings` from a
collected `Vec<(char, GlyphId)>`.

### Add to FontBuilder

```rust
builder.add_table(&cmap)?;
```

## 6. `GPOS`

Files: `src/tables/gpos.rs`, `generated/generated_gpos.rs`,
`src/tables/gpos/{value_record,builders}.rs`.

### Shape

```rust
pub struct Gpos {
    pub script_list:  OffsetMarker<ScriptList>,
    pub feature_list: OffsetMarker<FeatureList>,
    pub lookup_list:  OffsetMarker<PositionLookupList>,
    pub feature_variations: NullableOffsetMarker<FeatureVariations, WIDTH_32>,
}

pub type PositionLookupList = LookupList<PositionLookup>;

pub enum PositionLookup {
    Single(Lookup<SinglePos>),
    Pair(Lookup<PairPos>),
    Cursive(Lookup<CursivePosFormat1>),
    MarkToBase(Lookup<MarkBasePosFormat1>),
    MarkToLig(Lookup<MarkLigPosFormat1>),
    MarkToMark(Lookup<MarkMarkPosFormat1>),
    Contextual(Lookup<PositionSequenceContext>),
    ChainContextual(Lookup<PositionChainContext>),
    Extension(Lookup<ExtensionSubtable>),
}
```

Re-used layout types (`src/tables/layout.rs` + `generated_layout.rs`):

```rust
pub struct ScriptList  { pub script_records:  Vec<ScriptRecord> }
pub struct FeatureList { pub feature_records: Vec<FeatureRecord> }
pub struct LookupList<T> { pub lookups: Vec<T> }
pub struct Lookup<T>   {
    pub lookup_flag: LookupFlag,
    pub subtables: Vec<T>,
    /* optional mark_filtering_set */
}

impl<T: Default> Lookup<T> {
    pub fn new(lookup_flag: LookupFlag, subtables: Vec<T>) -> Self;
}
impl<T: Default> LookupList<T> { pub fn new(lookups: Vec<T>) -> Self; }
```

`Gpos: TopLevelTable` (`TAG = "GPOS"`), `Default`, `Clone`. Version is computed
from `feature_variations.is_some()` — you don't set it manually.

`ValueRecord` (`src/tables/gpos/value_record.rs`) is the GPOS value type. Use
the builder pattern:

```rust
let v = ValueRecord::new()
    .with_x_advance(120)
    .with_y_placement(-30)
    .with_x_advance_device(VariationIndex::new(0xff, 7));
```

`format()` is computed from which fields are set; `with_explicit_value_format`
lets you override.

`SinglePos`, `PairPos`, `CursivePosFormat1`, etc. each have `format_1` /
`format_2` / `new` constructors (see `generated_gpos.rs`). Helpers:

```rust
let cov: CoverageTable = [GlyphId16::new(2), GlyphId16::new(4)].into_iter().collect();
let cov2 = CoverageTable::format_1(vec![GlyphId16::new(2)]);
let cls: ClassDef  = [(GlyphId16::new(5), 0), (GlyphId16::new(6), 1)].into_iter().collect();

let sub  = SinglePos::format_1(cov, ValueRecord::new().with_x_advance(500));
let lkp  = Lookup::new(LookupFlag::default(), vec![sub]);
```

### Construct

From skrifa/read-fonts (the most common case for an "edit existing" tool):

```rust
let mut gpos: Gpos = font.gpos()?.to_owned_table();
```

From scratch:

```rust
let gpos = Gpos::new(
    ScriptList::new(vec![/* ScriptRecord::new(tag, Script::new(...)) */]),
    FeatureList::new(vec![/* FeatureRecord::new(Tag::new(b"kern"), Feature::new(...)) */]),
    LookupList::new(vec![PositionLookup::Single(lookup)]),
);
// gpos.feature_variations is a NullableOffsetMarker; leave it as default None
```

### Mutate

Walk through the deref of the `OffsetMarker`s:

```rust
// Append a kern lookup at index N
let new_lookup = PositionLookup::Pair(Lookup::new(LookupFlag::default(),
                                                  vec![my_pair_pos]));
let lookups = &mut gpos.lookup_list.as_mut().lookups;
let idx = lookups.len() as u16;
lookups.push(new_lookup);

// Hook it up to a feature record (kern):
gpos.feature_list.as_mut().feature_records.push(
    FeatureRecord::new(Tag::new(b"kern"),
                       Feature::new(None, vec![idx])),
);
```

For variation index remapping there is `RemapVariationIndices` (private trait
implemented for many subtable types) — you typically don't need this unless
you are rebuilding the IVS.

### Add to FontBuilder

```rust
builder.add_table(&gpos)?;
```

## 7. `GSUB`

File: `src/tables/gsub.rs`, `generated/generated_gsub.rs`.

Mirror image of GPOS. Same ScriptList/FeatureList/lookup_list layout.

### Shape

```rust
pub struct Gsub {
    pub script_list:  OffsetMarker<ScriptList>,
    pub feature_list: OffsetMarker<FeatureList>,
    pub lookup_list:  OffsetMarker<SubstitutionLookupList>,
    pub feature_variations: NullableOffsetMarker<FeatureVariations, WIDTH_32>,
}

pub enum SubstitutionLookup {
    Single(Lookup<SingleSubst>),
    Multiple(Lookup<MultipleSubstFormat1>),
    Alternate(Lookup<AlternateSubstFormat1>),
    Ligature(Lookup<LigatureSubstFormat1>),
    Contextual(Lookup<SubstitutionSequenceContext>),
    ChainContextual(Lookup<SubstitutionChainContext>),
    Extension(Lookup<ExtensionSubtable>),
    Reverse(Lookup<ReverseChainSingleSubstFormat1>),
}
```

`Gsub: TopLevelTable` (`TAG = "GSUB"`), `Default`, `Clone`. `Gsub::new(script,
feature, lookup)` exists; version is auto-computed.

Subtable enums and constructors (selected highlights from
`generated_gsub.rs`):

```rust
SingleSubstFormat1::new(coverage: CoverageTable, delta_glyph_id: i16)
SingleSubstFormat2::new(coverage: CoverageTable, substitute_glyph_ids: Vec<GlyphId16>)
MultipleSubstFormat1::new(coverage: CoverageTable, sequences: Vec<Sequence>)
Sequence::new(substitute_glyph_ids: Vec<GlyphId16>)
AlternateSubstFormat1::new(coverage, alternate_sets: Vec<AlternateSet>)
AlternateSet::new(alternate_glyph_ids: Vec<GlyphId16>)
LigatureSubstFormat1::new(coverage, ligature_sets: Vec<LigatureSet>)
LigatureSet::new(ligatures: Vec<Ligature>)
Ligature::new(ligature_glyph: GlyphId16, component_glyph_ids: Vec<GlyphId16>)
ReverseChainSingleSubstFormat1::new(/* large signature */)
```

`SingleSubst` is an enum of `Format1 | Format2`, with `SingleSubst::format_1`
and `SingleSubst::format_2` shorthand constructors.

### Construct, mutate, build

Identical pattern to GPOS — load with `font.gsub()?.to_owned_table()`, mutate
through the `OffsetMarker` derefs, register on `FontBuilder` via
`add_table(&gsub)?`.

Example: add a `liga` lookup mapping `[f, i] -> fi`.

```rust
let coverage = CoverageTable::format_1(vec![GlyphId16::new(F_GID)]);
let lig = Ligature::new(GlyphId16::new(FI_GID), vec![GlyphId16::new(I_GID)]);
let lig_set = LigatureSet::new(vec![lig]);
let sub = LigatureSubstFormat1::new(coverage, vec![lig_set]);
let lookup = Lookup::new(LookupFlag::default(), vec![sub]);
gsub.lookup_list.as_mut().lookups.push(SubstitutionLookup::Ligature(lookup));
// then push a FeatureRecord(b"liga") referencing this lookup index, like in GPOS.
```

## 8. End-to-end roundtrip example

```rust
use write_fonts::{
    FontBuilder, dump_table,
    from_obj::ToOwnedTable,
    tables::{hmtx::Hmtx, cmap::Cmap, hhea::Hhea, head::Head},
};
use write_fonts::read::{FontRef, TableProvider};   // "read" feature

fn rewrite(input: &[u8]) -> anyhow::Result<Vec<u8>> {
    // 1. parse
    let font = FontRef::new(input)?;

    // 2. pull tables we want to mutate into write-fonts owned form
    let mut hmtx: Hmtx = font.hmtx()?.to_owned_table();
    let mut cmap: Cmap = font.cmap()?.to_owned_table();
    let mut hhea: Hhea = font.hhea()?.to_owned_table();
    let mut head: Head = font.head()?.to_owned_table();

    // 3. mutate
    for m in &mut hmtx.h_metrics {
        m.advance = m.advance.saturating_add(50);   // widen every glyph by 50 units
    }
    cmap = Cmap::from_mappings(my_new_mapping())?;  // wholesale-replace cmap
    head.modified = font_types::LongDateTime::new(now_seconds_since_1904());

    // 4. assemble — overrides win over copy_missing_tables
    let bytes = FontBuilder::new()
        .add_table(&hmtx)?
        .add_table(&hhea)?
        .add_table(&cmap)?
        .add_table(&head)?
        .copy_missing_tables(font)
        .build();
    Ok(bytes)
}
```

Notes:

- Skrifa's `FontRef` is the same type as `read_fonts::FontRef` (skrifa
  re-exports it), so a `skrifa::FontRef::new(bytes)?` value can be passed
  straight to `copy_missing_tables`.
- `FontBuilder::build` takes `&mut self` and consumes the table map — call it
  once at the very end.
- Don't forget the dependency chain when mutating one table:
  - `hmtx` ↔ `hhea.number_of_h_metrics` (must equal `hmtx.h_metrics.len()`).
  - `glyf`/`loca` ↔ `head.index_to_loc_format` (set from `LocaFormat` returned
    by `GlyfLocaBuilder::build`).
  - Any change in glyph count propagates to `maxp.num_glyphs`, `hmtx`,
    optional `vmtx`, `loca`, `cmap` references…
